Build a one-page, mobile-first promotional landing page for iAlmas, intended for QR-code traffic at exhibitions and trade shows. The page must look premium, load fast, and convert quickly on mobile. The goal is to attract business visitors, establish credibility in seconds, showcase trusted vendors, and drive direct contact.

Do not build a generic corporate homepage. Build a focused exhibition landing page optimized for users who scan a QR code and decide quickly whether to engage.

Project intent:
- One-page landing page only
- Promotional and trust-building
- Mobile-first, but polished on desktop too
- Enterprise cybersecurity visual language
- Arabic-ready structure
- High-conversion CTA placement
- Premium, clean, dark UI based on the following design system

Typography:

UI direction:

- Subtle animation only
- No noisy effects
- No generic template feel

1. who iAlmas is
2. what iAlmas offers
3. which vendors iAlmas works with
4. how to contact the team immediately

Core UX requirements:
- Mobile-first
- Hero must explain the company immediately
- CTA visible early

- Contact section must be frictionless
- Footer must include useful links
- Keep copy short, promotional, and credible

CONTENT STYLE RULES

- Write concise promotional copy
- Do not use long paragraphs
- Avoid hype and vague startup language
- Keep the tone serious, polished, and enterprise-focused
- Make the page suitable for cybersecurity buyers and exhibition visitors
- Use real labels and real vendor names
- Do not invent awards, metrics, or claims
- Do not add fake testimonials
- Do not add blog, pricing, login, careers, or unrelated pages

RESPONSIVENESS

- Mobile first
- Optimized for QR traffic on phones
- Thumb-friendly buttons
- Strong spacing and readability on small screens
- Vendor tiles must stack or wrap elegantly
- Hero must remain strong without requiring excessive scrolling
- Contact section must remain accessible and easy to fill

ARABIC-READY REQUIREMENTS

Prepare the implementation so Arabic can be added later without major refactor:
- centralize text in content objects/constants
- avoid layout assumptions that break with longer text
- structure components to support RTL later
- use typography and spacing that can tolerate bilingual content
- do not implement full RTL now unless needed, but keep the architecture ready

TECHNICAL EXPECTATIONS

- Build clean, maintainable components
- Prefer a componentized structure
- Keep content easy to edit
- Use a data-driven approach for vendors, solutions, links, and CTA items
- Preserve performance
- Avoid unnecessary libraries
- Use tasteful animation only
- Use accessible semantic HTML
- Include section IDs for anchor links
